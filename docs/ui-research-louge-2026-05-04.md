# 楼阁 UI 竞品与视觉方向梳理 - 2026-05-04

## 观察对象

- Linear：克制深色、精确间距、细边框、低噪声、高信息密度；高级感主要来自秩序和层级，不靠大量装饰。
- Cursor：极简黑白基底，局部高光，强标题，强调“新范式”的产品感；按钮和模块都很干净。
- Framer：大留白、强视觉展示、局部全息/渐变效果；惊艳点集中在少数核心区域，而不是全页平均发光。
- Replika / Character.AI / Janitor AI 类：角色陪伴产品需要“人味”：头像、卡片、对话气泡、关系感、柔和情绪线索，但不能做得廉价或过度二次元。
- Awwwards / Dribbble 深色 SaaS 趋势：黑曜石底 + 玻璃材质 + 高质量渐变边缘 + 大字号 + subtle motion。趋势是“少而准”，不是满屏霓虹。

## 对楼阁的结论

楼阁不该像普通后台，也不该像喧闹的娱乐平台。它应该像“夜里开着灯的私人角色馆”：

1. 第一眼：黑曜石空间、月光暖边、少量紫调，简约高级。
2. 浏览角色：像在看一组收藏卡/人物档案，而不是列表工具。
3. 聊天：气泡要舒服，输入区要像“安静写信”，角色侧栏要有陪伴感。
4. 管理/表单：仍保持效率和清晰，不牺牲可读性。
5. 动效：只保留轻微上浮、光边、呼吸，不做花哨粒子。

## 设计关键词

- Obsidian Gallery / 黑曜石角色馆
- Moonlit Companion / 月下陪伴
- Editorial SaaS / 杂志式产品官网排版
- Quiet Luxury / 安静奢华

## 实施原则

- CSS 为主，兼容现有 EJS class。
- 不把所有元素都做成紫色；紫色只作为品牌光，暖金作为记忆点。
- 统一圆角、边框、阴影、按钮、输入框、卡片、聊天气泡。
- 首页/公开角色/详情/聊天/工作台/后台共享一个高级视觉语言。
- 移动端保持 44px 点击面积和更强可读性。
